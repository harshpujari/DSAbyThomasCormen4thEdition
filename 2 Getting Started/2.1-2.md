# SUM-ARRAY(A, n)

```text
1 sum = 0
2 for i = 1 to n
3     sum = sum + A[i]
4 return sum
```

## Loop Invariant
At the start of each iteration of the for loop with index `i`, the variable `sum` contains the sum of the first `i-1` elements of `A`, that is:

`sum = A[1] + A[2] + ... + A[i-1]`

## Initialization
Before the first iteration, `i = 1` and `sum = 0`.
The sum of the first `i-1 = 0` elements is `0`, so the invariant holds.

## Maintenance
Assume the invariant holds at the start of iteration `i`.
Then `sum = A[1] + ... + A[i-1]`.
The body executes `sum = sum + A[i]`, so after the update:

`sum = A[1] + ... + A[i-1] + A[i]`

At the start of the next iteration, the loop index is `i + 1`, and the invariant now says `sum` contains the sum of the first `(i + 1) - 1 = i` elements.
Thus the invariant is maintained.

## Termination
The loop finishes after the iteration with `i = n`.
At that point, the invariant says `sum` contains `A[1] + ... + A[n]`.
The procedure returns `sum`, so it returns the sum of all `n` elements of `A`.

## Conclusion
By initialization, maintenance, and termination of the loop invariant, `SUM-ARRAY` returns the correct total for `A[1..n]`.
